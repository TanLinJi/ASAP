ASAP: Adaptive Spherical Anomaly-Guided Purification Against LiDAR Point Cloud Attacks

0. 项目元信息

投稿目标

会议：ICASSP 2026（IEEE International Conference on Acoustics, Speech and Signal Processing）

篇幅：4 页正文 + 1 页参考文献（IEEE 双栏）

预计 deadline：2026 年 9 月（abstract 通常早 1 周）

研究主题：LiDAR point cloud adversarial attacks and defenses for 3D object detection

核心问题：在点扰动（point perturbation）和点注入（point injection）攻击下，提高 3D object detector 的对抗鲁棒性，同时尽量保持 clean mAP 与推理速度。

领域 track：Image, Video & Multimedia Signal Processing / 3D Vision / Multimedia Security

工程基础

启发来源：LiDAR-SPD（ICASSP 2025，UESTC）提出 SPU 级净化范式

实现路线：直接实现 ASAP，不复现作者未公开的完整 LiDAR-SPD 代码

3D 检测框架：OpenPCDet_uni

扩散参考：DiffPure（NVlabs），PointDP（Sun 2023）

命名约定

正文称 ASAP（Adaptive Spherical Anomaly-Guided Purification）。LiDAR-SPD 仅作为已有 SPU 级净化范式与 reported baseline 引用，不作为本文必须复现的代码基线。

2. LiDAR 点云对抗攻击与 SPU-级净化（受 LiDAR-SPD 启发）

2.1 对抗攻击问题设定

给定被对抗攻击污染的 LiDAR 点云场景 Ŝ = S + T + P̂，其中 T 表示攻击者对真实点的点扰动（point perturbation），P̂ 表示攻击者额外加入的点注入（point injection）。目标是构建一个不依赖具体检测器的前处理净化模块 g，使得 g(Ŝ) ≈ S，从而让 3D detector 在攻击场景下仍保持 D(g(Ŝ)) ≈ D(S)。

2.2 SPU 范式如何应对点云攻击

LiDAR-SPD 的关键贡献是把大规模 LiDAR scene 划分为局部 Spherical Purification Unit (SPU)，再在 SPU 级别用球形投影处理注入攻击、用扩散净化处理扰动攻击。ASAP 采用这种球形局部单元思想，但不依赖原作者代码，而是在其范式上重新设计自适应半径、异常打分与选择性扩散，使防御更适合真实 LiDAR 点云对抗场景。

无防御时 ASR 普遍在 70~95%；LiDAR-SPD 报告值显示 SPU 级净化有潜力将 ASR 降到 14~47%。本文将这些数值作为 reported baseline，而不是复现目标。

叙事边界

本文不是 LiDAR-SPD 的工程复现报告，而是提出一个新的 ASAP SPU-level adversarial purification framework。LiDAR-SPD 负责证明 SPU 级净化可用于 LiDAR 点云对抗防御；ASAP 负责解决该类防御在速度、密度适配和干净精度上的系统性问题。

3. LiDAR 点云对抗防御中的现有局限性

S1 推理慢

KITTI 一帧约 12 万点，固定半径会切出 几百到上千个 SPU。如果为了抵御点扰动和点注入攻击而让每个 SPU 都走 VP-SDE 反向积分，推理成本会迅速放大。

证据

已有 diffusion purification 工作通常计算开销高；LiDAR-SPD 报告值中未给出 fps 或 latency。

S2 半径全局固定

固定 r1, r2 难以适配真实 LiDAR 的非均匀采样。点密度随距离 d 近似按 1/d² 衰减——5m 处一个 SPU 可能含 200~500 点，50m 处可能只剩 3~10 点。

代价

远处 SPU 点太少时扩散模型无法稳定生成，可能放大噪声。

S3 无差别净化

如果所有 SPU 一视同仁地强制扩散，那么 clean scene 上也会被改动。这会降低干净精度，并且把计算预算浪费在明显正常的局部区域。

证据

LiDAR-SPD 报告的主表只关注 ASR，没有系统报告净化后 clean mAP；这正是 ASAP 要补上的指标。

本工作的中心思想

S1 + S2 + S3 是耦合的。如果我们能识别"哪些 SPU 可能被对抗攻击污染"并且"按密度适配半径"，那么：

  1. 仅对可疑 SPU 扩散 → 速度暴增（解决 S1）
  2. 不动干净 SPU → clean mAP 不掉（解决 S3）
  3. 密度自适应半径 → 稀疏区不再失效（解决 S2）

三个改进同源、互相增益，构成"面向点云对抗攻击的高效鲁棒净化"完整故事。

4. ASAP 对抗攻击防御框架总览

三个 NEW 模块对应三个局限

M1 Adaptive Radius → 解决 S2

M2 Anomaly Scorer → 决定哪些 SPU 进 M3

M3 Selective Diffusion → 解决 S1 和 S3

Spherical Projection 保留物理遮挡建模思想——它是 SPU 级净化应对注入攻击的关键几何先验；ASAP 在此基础上加入自适应半径与选择性扩散。

5. 模块一：自适应半径估计

5.1 设计动机

LiDAR 点云的局部密度 ρ(p) 强烈依赖于距离 d = ‖p‖₂ 和入射角 θ。最简单的物理近似：

ρ(p) ∝ cos(θ) / d²

固定 r1 会让远处 SPU 退化（点太少 → 扩散崩坏），近处 SPU 浪费算力（点太多）。

5.2 计算方法

候选 A：KNN-based（推荐）

对每个候选球心 p，找第 k 近邻距离 d_k(p)（k=16）。

r1(p) = α · d_k(p), α ≈ 2.0

优点：scipy KDTree 直接给，O(N log N)。

缺点：远处不足 k 个点时需 fallback。

候选 B：Range-based（备用）

用 LiDAR 物理参数（垂直角分辨率 Δφ、水平 Δθ）：

r1(p) = α · d · max(Δφ, Δθ)

优点：纯几何，O(N)。

缺点：依赖 sensor specs（KITTI Δφ≈0.4°, Δθ≈0.08°）。

推荐

主用 A：实现简单、不依赖 sensor 参数、跨数据集通用。B 作 ablation 对比。

5.3 边界处理

  • 设上下界：r1 ∈ [0.10m, 0.40m]
  • r2 联动：r2 = 0.67 · r1（参考 0.10/0.15 的经验比例）

6. 模块二：异常打分器

6.1 设计哲学

不训练深度异常检测器（增加训练负担、需伪标签）。改用无需训练的几何统计量组合 score——快、可解释、不引入额外参数。

6.2 三个候选特征

f1 Compactness 紧致度

干净物体的局部点云贴合 2D 流形（车身、地面）。注入点散乱在 3D 体素里。

f1 = σ(d_to_centroid)

σ 大 → 异常。

f2 Anisotropy 各向异性

对 SPU 内点做 PCA 得特征值 λ₁≥λ₂≥λ₃：

f2 = λ₃ / (λ₁+λ₂+λ₃)

合法表面 f2≈0，注入团块 f2→1/3。

f3 vMF Kappa 方向集中度

SPU 内点归一化到单位球，估计 vMF 集中参数 κ̂：

f3 = -log(κ̂)

κ̂ 大 → 朝向集中（自然表面）；小 → 散乱 → 异常。

6.3 综合 score 与阈值

三个特征 z-score 标准化后线性加权：

score = w₁·z(f1) + w₂·z(f2) + w₃·z(f3)

权重 w 通过 KITTI 训练集做二分类逻辑回归学到：

  1. clean 训练图 → SPU 标 label=0
  2. attacked 训练图 → SPU 与攻击点 IoU>0.5 标 label=1
  3. [f1,f2,f3] 做 logistic regression，得 w₁,w₂,w₃

阈值 τ 通过 ROC 曲线上 Youden 指数（J = TPR − FPR 最大）确定。

写作技巧

逻辑回归只 3 参数，避免审稿人指责"加了可训练检测器"。论文可写成"weights determined by simple grid search"——更轻。

7. 模块三：选择性扩散

7.1 核心改动

对每个 SPU：

  • 若 score > τ：进入 VP-SDE / PointDP-style 扩散（forward → backward with local feature z）
  • 若 score ≤ τ：保持原始点云不动（bypass）

7.2 进一步加速：批处理

uniform SPU diffusion 会对每个 SPU 单独调用 sdeint。ASAP 把所有可疑 SPU 在点维度 pad 到相同 N_max，批处理送入扩散网络：

X_batch = pad([SPU₁, ..., SPU_k]) → shape (k, N_max, 3)
X_purified = sdeint_batch(X_batch, z_batch)

预期在 RTX 4090 上单帧推理从 ~30s 降到 ~1-3s（依实测）。

7.3 局部特征 z 的复用

使用 PointNet++ 风格的局部编码器为 SPU 提取 z。ASAP 只在可疑 SPU 上计算 z，从而同时减少 encoder forward 与 reverse-SDE 的开销。

8. 训练目标

ASAP 使用 PointDP / DiffPure 风格的扩散净化损失，不额外堆复杂监督项。本文的创新点集中在 SPU 构造、异常门控与批处理推理，而不是重新发明 diffusion loss。

扩散净化基础损失

(1) Score matching loss（扩散主损失）

L_sm = E[‖ε_θ(x_t, t, z) − ε‖²]

(2) Local purification loss

L_local = Hausdorff(p, p') + ‖z−z'‖² / exp(z·z')

新加（可选）

Anomaly score 监督：

L_score = BCE(σ(w·f), y_attack)

注：独立训练 3-参数模型，不与扩散联合训练。

13. 风险与备选方案

风险 1：clean mAP 没回升

表现：ASAP 的 clean mAP 与 no defense 差距 > 1pp，或相对 uniform SPU diffusion 没有明显优势。

原因：异常打分器假阳性率太高，扩散仍过度。

备选：

  1. 提高阈值 τ → 减少扩散，clean mAP ↑ 但 ASR ↑ → 调权衡。
  2. 把卖点改为 "5-20× 加速"，一样能投。

风险 2：异常打分器被指 trivial

备选：

  1. f1/f2/f3 升级为 vMF kappa + DGCNN feature 方差。
  2. ablation 证明组合显著优于单特征。
  3. 用 ROC 证明 score 本身是 strong attack detector（AUC>0.85），独立报告为 by-product。

风险 3：4 页装不下

备选：

  1. 合并 M1+M2 描述（共 0.5 页 method）。
  2. Table I+II+III 压成一张 3 列大表。
  3. 所有 ablation 进 supplementary。
  4. 砍掉 M1 自适应半径，只留 M2+M3，题目改为 "Anomaly-Gated Diffusion for Efficient Defense Against LiDAR Point Cloud Adversarial Attacks"。

风险 4：审稿人指 Ada3Diff 已经"adaptive"

表现：审稿人评 "The adaptive idea is already explored in Ada3Diff (MM 2023). What's new?"

预案（必须在 intro 第一段就反击）：

  1. 明确护城河：Ada3Diff 是 point-level adaptive timestep on clean classification；ASAP 是 SPU-level adaptive radius + binary gate on raw LiDAR detection scenes。两个 "adaptive" 不在同一层级。
  2. 实证差异：把 Ada3Diff 的 plane-fit distortion estimator 强行套到 KITTI 上跑，证明它在 sparse LiDAR scenes 上明显劣于 ASAP 的 SPU-级机制（plane-fitting 在稀疏区无法稳定拟合）。
  3. 引用顺位：Intro 第二段就引用 Ada3Diff，main figure 的 baseline 列里出现它——把"知情且超越"的信号摆在最前。

风险 5：与 LiDAR-SPD reported 数字可比性不足

表现：审稿人评 "The comparison to LiDAR-SPD is not fully controlled since only reported numbers are used."

预案：

  1. 明确表中 LiDAR-SPD 为 reported baseline，不把它包装成本文实现结果。
  2. 使用与 LiDAR-SPD reported setting 对齐的 KITTI val split + detector 配置，并在实验设置中逐项列出 detector checkpoint、split 与 attack budget。
  3. Ablation A2 里"ASAP without selective gating + uniform radius" 等价于一个 SPU-级 uniform diffusion baseline——作为本文内部公平对照。
总体风险评估

采取"自主提出 ASAP"叙事后，主要风险是 Ada3Diff 思路重叠（已有 4 维护城河应对）和 clean mAP 不回升（可调阈值 τ）。

缓解因素：

  • 3 个新模块（M1/M2/M3）本身是无需训练的几何方法——失败概率低
  • 扩散组件可直接复用 PointDP 的开源 score net
  • OpenPCDet_uni 已有完整 detector 训练/评测 pipeline
  • 护城河（SPU 级 + 物理一致性 + LiDAR-specific）相对 Ada3Diff 清晰
  • 不需要复现 LiDAR-SPD，节省 W1-W4 大量工作量

预估出稿概率：65-75%。

14.1 顶层 pipeline

class ASAP(SphericalPurification):
    def __init__(self, alpha=2.0, beta=0.67,
                 r1_min=0.10, r1_max=0.40,
                 tau=0.60, knn_k=16):
        self.alpha = alpha
        self.beta = beta
        self.r1_min, self.r1_max = r1_min, r1_max
        self.tau = tau
        self.knn_k = knn_k
        self.diff_model = load_pretrained_diffusion()    # PointDP-style score net
        self.encoder    = PointNetPP_local_encoder()      # local feature encoder E

    def __call__(self, point_cloud):
        radii = self.adaptive_radius(point_cloud)        # Module 1
        units = self.build_spu(point_cloud, radii)
        units = self.spherical_projection(units)            # physics-grounded occlusion prior

        scores     = self.anomaly_score(units)            # Module 2
        suspicious = scores > self.tau

        purified = self.selective_diffuse(units, suspicious) # Module 3
        return unify(purified)

14.2 Module 1: Adaptive Radius (KNN-based)

def adaptive_radius(self, points):
    # points: (N, 3)
    kdtree = KDTree(points)
    dist, _ = kdtree.query(points, k=self.knn_k)
    d_k = dist[:, -1]                        # 第 k 近邻距离
    r1 = self.alpha * d_k
    r1 = np.clip(r1, self.r1_min, self.r1_max)
    return r1                                 # 每点一个 r1 候选

14.3 Module 2: Anomaly Scorer

def anomaly_score(self, units):
    feats = []
    for (center, idx) in units:
        pts = points[idx]                       # (M, 3)
        # f1: compactness
        f1 = np.std(np.linalg.norm(pts - pts.mean(0), axis=1))
        # f2: anisotropy via PCA
        cov = np.cov(pts.T)
        evals = np.linalg.eigvalsh(cov)         # λ₁≤λ₂≤λ₃
        f2 = evals[0] / (evals.sum() + 1e-9)
        # f3: vMF kappa
        dirs  = (pts - center) / (np.linalg.norm(pts - center, axis=1, keepdims=True) + 1e-9)
        r_bar = np.linalg.norm(dirs.mean(0))
        kappa = r_bar * (3 - r_bar**2) / (1 - r_bar**2 + 1e-9)
        f3 = -np.log(kappa + 1e-9)
        feats.append([f1, f2, f3])
    feats = np.array(feats)
    feats = (feats - feats.mean(0)) / (feats.std(0) + 1e-9)  # z-score
    score = feats @ self.w                       # 训练好的 w (3,)
    return score

14.4 Module 3: Selective Diffusion (batched)

def selective_diffuse(self, units, suspicious):
    out = []
    suspicious_idx = np.where(suspicious)[0]
    # pad to common length and batch
    pts_list = [points[units[i][1]] for i in suspicious_idx]
    pts_batch, mask = pad_and_stack(pts_list)       # (k, N_max, 3)
    z_batch = self.encoder(pts_batch, mask)        # (k, D)

    pts_purified = self.sdeint_batch(
        pts_batch, z_batch, mask,
        t_start=0.1, t_end=0.0
    )                                            # short reverse-SDE horizon

    # merge results
    for i, unit in enumerate(units):
        if suspicious[i]:
            j = np.where(suspicious_idx == i)[0][0]
            out.append(unpad(pts_purified[j], mask[j]))
        else:
            out.append(points[unit[1]])           # bypass
    return out
